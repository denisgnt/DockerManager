# Docker Manager

Современный веб-интерфейс для управления Docker контейнерами через Docker API.

## 🚀 Возможности

- ✅ Просмотр списка всех Docker контейнеров (список/плитки)
- ✅ Управление контейнерами (старт, остановка, перезапуск, удаление)
- ✅ Просмотр логов контейнеров в реальном времени через WebSocket
- ✅ **Поддержка цветных логов (ANSI color codes)** - автоматическое отображение bash-цветов
- ✅ **Выполнение скриптов rebuild** - автоматическое сопоставление и запуск скриптов
- ✅ Подробная информация о контейнерах
- ✅ Сортировка, поиск и фильтрация
- ✅ Автоматическое обновление списка контейнеров
- ✅ Современный интерфейс на базе Material UI 7
- ✅ Темная тема оформления
- ✅ Адаптивный дизайн

## 🛠 Технологии

### Frontend
- **React 19.2** - библиотека для создания пользовательских интерфейсов
- **React Compiler** - автоматическая оптимизация компонентов
- **Zustand** - управление глобальным состоянием приложения
- **Vite 7.1** - современный сборщик проектов
- **Material UI 7.3** - библиотека компонентов
- **Socket.IO Client 4.8** - для WebSocket соединений
- **Axios** - для HTTP запросов
- **ansi-to-react** - для отображения ANSI цветов

### Backend
- **Node.js** - серверная платформа
- **Express 5.1** - веб-фреймворк
- **Socket.IO 4.8** - для WebSocket сервера
- **Axios** - для взаимодействия с Docker API

## 📋 Требования

- Node.js 18+ и yarn
- **ИЛИ**
- Docker и Docker Compose

## 🚢 Запуск в Docker (рекомендуется)

Для production запуска используйте Docker Compose:

```bash
# Настройте .env файл
cp .env.docker .env
# Отредактируйте .env под ваши нужды

# Запустите приложение
docker-compose up -d
```

Приложение будет доступно на `http://localhost:3001`

📖 Подробнее см. [DOCKER_DEPLOYMENT.md](./DOCKER_DEPLOYMENT.md)

## 🔧 Локальная разработка

1. Клонируйте репозиторий или перейдите в директорию проекта:

```bash
cd DockerManager
```

2. Установите зависимости:

```bash
npm install
```

3. Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

4. Настройте переменные окружения в `.env`:

```env
VITE_DOCKER_API_HOST=10.174.18.242
VITE_DOCKER_API_PORT=2375
VITE_PORT=3001
VITE_BACKEND_PORT=5001
```

См. подробную документацию в [ENV_VARIABLES.md](./ENV_VARIABLES.md)

3. Если необходимо, измените адрес Docker API в файле `server/index.js`:

```javascript
const DOCKER_HOST = 'http://10.174.18.242:2375';
```

## 🚀 Запуск

### Режим разработки

Запустите приложение в режиме разработки (одновременно запускаются frontend и backend):

```bash
npm run dev
```

- Frontend будет доступен по адресу: `http://localhost:3000`
- Backend API будет доступен по адресу: `http://localhost:5000`

### Отдельный запуск компонентов

Запуск только backend сервера:


```bash

```

Создание продакшн сборки:

```bash
npm run build
```

Предварительный просмотр продакшн сборки:

```bash
npm run preview
```

## 🎯 API Endpoints

### Backend API

- `GET /api/containers` - Получить список всех контейнеров
- `GET /api/containers/:id` - Получить информацию о конкретном контейнере
- `POST /api/containers/:id/start` - Запустить контейнер
- `POST /api/containers/:id/stop` - Остановить контейнер
- `POST /api/containers/:id/restart` - Перезапустить контейнер
- `DELETE /api/containers/:id` - Удалить контейнер
- `GET /api/containers/:id/logs` - Получить логи контейнера
- `GET /api/containers/:id/stats` - Получить статистику контейнера
- `GET /api/info` - Получить информацию о Docker

### WebSocket Events

- `subscribe-logs` - Подписаться на логи контейнера
- `unsubscribe-logs` - Отписаться от логов контейнера
- `log-data` - Получение данных логов
- `log-error` - Ошибка при получении логов

## 🎨 Функционал интерфейса

### Список контейнеров

- Отображение всех контейнеров с информацией:
  - Имя контейнера
  - Образ
  - ID (короткий)
  - Статус (с цветовой индикацией)
  - Порты
  - Дата создания

### Управление контейнерами

- **Запуск** - зеленая кнопка с иконкой Play
- **Остановка** - красная кнопка с иконкой Stop
- **Перезапуск** - оранжевая кнопка с иконкой Restart
- **Удаление** - красная кнопка с иконкой Delete (с подтверждением)
- **Просмотр логов** - синяя кнопка с иконкой Eye

### Просмотр логов

- Логи в реальном времени через WebSocket
- Автоматическая прокрутка к последней записи
- Индикатор подключения
- Возможность очистки логов
- Монохромный стиль в стиле терминала

## ⚙️ Настройка

### Изменение порта Docker API

Отредактируйте `server/index.js`:

```javascript
const DOCKER_HOST = 'http://your-docker-host:2375';
```

### Изменение портов приложения

В `vite.config.js` измените порт frontend:

```javascript
server: {
  port: 3000,  // измените на нужный порт
}
```

В `server/index.js` измените порт backend:

```javascript
const PORT = process.env.PORT || 5000;  // измените на нужный порт
```

### Настройка интервала обновления

В `src/App.jsx` измените интервал автообновления:

```javascript
const interval = setInterval(fetchContainers, 5000); // 5 секунд
```

## 🔒 Безопасность

⚠️ **ВАЖНО**: Данное приложение использует незащищенное соединение с Docker API (порт 2375). 

Для использования в продакшене рекомендуется:

1. Использовать Docker API с TLS (порт 2376)
2. Настроить авторизацию в приложении
3. Использовать HTTPS для веб-интерфейса
4. Ограничить доступ к Docker API через файрвол
5. Использовать VPN для удаленного доступа

Подробнее о настройке безопасного соединения см. в файле `docker-api-setup.md`.

## 🐛 Решение проблем

### Не удается подключиться к Docker API

1. Убедитесь, что Docker API доступен:
   ```bash
   curl http://10.174.18.242:2375/version
   ```

2. Проверьте настройки файрвола на Docker хосте

3. Убедитесь, что Docker слушает на порту 2375:
   ```bash
   sudo netstat -tlnp | grep 2375
   ```

### Логи не отображаются в реальном времени

1. Проверьте консоль браузера на наличие ошибок WebSocket
2. Убедитесь, что backend сервер запущен
3. Проверьте, что контейнер генерирует логи

### Порт уже используется

Если порты 3000 или 5000 заняты, измените их в конфигурации (см. раздел "Настройка").
